# 릴리즈 페이지

## 릴리즈 원칙

이 프로젝트의 릴리즈는 차량 내 어린이 또는 애완동물 감지와 알림이라는 안전 관련 기능을 포함하므로, 기능 완료뿐 아니라 테스트 증거, 알려진 제한사항, 개인정보 보호 점검을 함께 공개한다.

## 버전 정책

- 버전 형식: `v<major>.<minor>.<patch>`
- `major`: 알림 정책, 데이터 모델, 외부 API 등 호환성이 깨지는 변경
- `minor`: 새 기능 또는 주요 정책 추가
- `patch`: 버그 수정, 문서 보완, 내부 품질 개선

## 릴리즈 브랜치 흐름

1. `release/v<major>.<minor>.<patch>` 브랜치를 생성한다.
2. 요구사항, 테스트, 변경 이력을 동기화한다.
3. GitHub Actions 검증을 통과시킨다.
4. 릴리즈 PR을 `main`에 병합한다.
5. `v<major>.<minor>.<patch>` 태그를 생성한다.
6. GitHub Releases에 릴리즈 노트를 게시한다.

## 릴리즈 체크리스트

- [ ] `requirements.md`의 Must 요구사항이 테스트로 연결되어 있다.
- [ ] `pytest` 전체 테스트가 통과한다.
- [ ] 커버리지 기준을 만족한다.
- [ ] `ruff check .`가 통과한다.
- [ ] `ruff format --check .`가 통과한다.
- [ ] `mypy src`가 통과한다.
- [ ] 원본 영상, 음성, 연락처 원문, 위치 세부값 저장 여부를 점검했다.
- [ ] 오탐, 미탐, 센서 제약, 환경 제약을 릴리즈 노트에 기록했다.
- [ ] 알림 실패와 재시도 정책을 검토했다.
